   • Real-time object detection from RGB/depth images
   • Confidence filtering and NMS
   • Bounding box to spatial conversion
   • Detection caching for performance
   • Statistics tracking
   
   Key Features:
   - Supports multiple YOLO model sizes (nano, small, medium, large)
   - Configurable confidence threshold and IoU threshold
   - GPU acceleration support
   - Detection visualization
   - Spatial information extraction (distance, angle)
   
   Classes:
   ├─ YOLODetector: Single YOLO model wrapper
   └─ YOLOEnsemble: Multiple YOLO models for ensemble predictions

2. yolo_cnn_fusion.py (~350 lines)
   ──────────────────────────────
   • CNNYOLOFusion: Fuses CNN depth + YOLO detection features
   • Dual-modality perception system
   • Multiple fusion modes (concatenation, attention, weighted)
   
   Key Features:
   - Extracts CNN features from depth images (128D)
   - Extracts YOLO features from RGB detections (18D)
   - Combines into unified representation (146D)
   - Attention-based feature weighting
   
   Classes:
   ├─ CNNYOLOFusion: Core fusion mechanism
   └─ FusionObservationBuilder: Builds complete 160D observations

3. yolo_video_analyzer.py (~360 lines)

   • Real-time YOLO overlay visualization
   • State information display
   • Detection and trajectory statistics
   
   Key Features:
   - MP4 video recording with YOLO detections
   - Frame-by-frame state logging
   - Performance statistics generation
   - Automatic directory management
   
   Classes:
   ├─ YOLOVideoAnalyzer: Main video recording class
   └─ VideoFrameBuffer: Efficient frame buffering

4. yolo_setup.py (~220 lines)
   ──────────────────────────

   • Dependency management
   • Model downloading

🔧 INTEGRATION STEPS
═══════════════════════════════════════════════════════════════════════════════

Step 1: Installation
──────────────────
python yolo_setup.py
# Or manually:
pip install ultralytics opencv-python torch torchvision

Step 2: Import in Training Script
─────────────────────────────────
from yolo_detector import YOLODetector
from yolo_cnn_fusion import CNNYOLOFusion, FusionObservationBuilder
from yolo_video_analyzer import YOLOVideoAnalyzer

Step 3: Initialize in Training Loop
───────────────────────────────────
# Initialize YOLO detector
yolo_detector = YOLODetector(
    model_name='yolov8n',  # Lightweight for real-time
    confidence_threshold=0.5,
    iou_threshold=0.45,
    device='cuda' if torch.cuda.is_available() else 'cpu'
)

# Create fusion module
fusion_module = CNNYOLOFusion(
    depth_cnn=env.depth_extractor,
    yolo_detector=yolo_detector,
    fusion_mode='concatenate'
)

# Build observations with fusion
obs_builder = FusionObservationBuilder(fusion_module)

Step 4: Use in Training Loop
────────────────────────────
# Get images from environment
depth_image = env.render_depth()  # 64x64
rgb_image = env.render_rgb()      # HxWx3

# Build fused observation (160D)
fused_obs = obs_builder.build(
    pos=obs[:3],
    vel=obs[3:6],
    goal_pos=env.goal_pos,
    depth_image=depth_image,
    rgb_image=rgb_image
)

# Use fused observation for policy
action = agent.select_action(fused_obs)

Step 5: Optional Video Recording
────────────────────────────────
video_recorder = YOLOVideoAnalyzer(
    output_dir='yolo_videos',
    fps=30,
    frame_size=(1280, 720),
    enable_yolo_viz=True
)

# In training loop:
video_recorder.write_frame(
    rgb_image,
    detections=yolo_detector.detect(rgb_image),
    state_info={
        'position': obs[:3],
        'velocity': obs[3:6],
        'goal_distance': np.linalg.norm(env.goal_pos - obs[:3]),
        'episode': episode_num,
        'collision': env.collision_detected
    }
)

# After training:
stats = video_recorder.finalize()


⚡ PERFORMANCE CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════════

Model Selection:
├─ YOLOv8n (nano):    ~5-10ms   per frame (GPU), ideal for real-time
├─ YOLOv8s (small):   ~10-20ms  per frame (GPU), good balance
├─ YOLOv8m (medium):  ~20-40ms  per frame (GPU), better accuracy
└─ YOLOv8l (large):   ~40-80ms  per frame (GPU), highest accuracy

Recommended: YOLOv8n for training, YOLOv8s for evaluation

GPU vs CPU:
├─ GPU (CUDA): 5-10x faster inference
├─ CPU:        Slower but no VRAM limitations
└─ Recommendation: Use GPU for training, CPU fallback available

Optimization Tips:
├─ Enable model caching (reduces redundant inference)
├─ Batch images together if possible
├─ Use frame buffering for video recording
├─ Consider model quantization for edge deployment
└─ Profile inference time with: agent.profile_inference()

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: "ModuleNotFoundError: No module named 'ultralytics'"
Solution: pip install ultralytics

Issue: "CUDA out of memory"
Solution: Use YOLOv8n (nano) model or reduce batch size

Issue: "YOLO detections are empty"
Solution: Check RGB image brightness/contrast, increase confidence_threshold

Issue: "Video recording is slow"
Solution: Reduce frame size or disable YOLO visualization

Issue: "Fusion observation NaN values"
Solution: Check image normalization, ensure all inputs are valid

Issue: "Training takes too long"
Solution: Disable video recording, use faster YOLO model, enable GPU
