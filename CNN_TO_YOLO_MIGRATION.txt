🎯 MIGRATION OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

Transition from:
  • SimplDepthCNN (custom 3-layer CNN, 128D output)
  • DepthFeatureExtractor (custom depth processing wrapper)
  
To:
  • YOLOv8 (pre-trained real-time object detection)
  • YOLODepthDetector (YOLO-based spatial feature extraction)
  • YOLODepthFeatureExtractor (wrapper for seamless integration)

Key Benefits:
  ✓ Pre-trained on 80 COCO classes (obstacles, goals, people, vehicles, etc.)
  ✓ Real-time inference (5-10ms for nano model)
  ✓ Semantic understanding (WHAT is detected, not just patterns)
  ✓ No training required (zero-shot)
  ✓ 64D spatial features (compact vs 128D CNN)
  ✓ Easier to understand and debug detections
  ✓ Scalable to multiple object types


📊 FEATURE COMPARISON
═══════════════════════════════════════════════════════════════════════════════

SimplDepthCNN (Removed):
├─ Architecture: 3 Conv layers + 2 MLP
├─ Input: 64x64 depth image (grayscale)
├─ Output: 128D feature vector
├─ Properties:
│  ├─ Learned representations (black box)
│  ├─ Requires training
│  ├─ Fast inference (~2-3ms)
│  └─ Only understands depth patterns
└─ Limitations:
   ├─ No semantic understanding
   ├─ Must be trained with UAV task
   ├─ Difficult to interpret
   └─ Cannot transfer to other environments

YOLOv8 (New):
├─ Architecture: Vision Transformer backbone + YOLO head
├─ Input: Any RGB image (64x64 to 1280x1280)
├─ Output: 64D spatial feature vector
├─ Properties:
│  ├─ Pre-trained on COCO dataset
│  ├─ Zero-shot (no training needed)
│  ├─ Fast inference (5-10ms for nano)
│  └─ Detects COCO classes (80 types)
├─ Advantages:
│  ├─ Semantic understanding (person, car, bottle, etc.)
│  ├─ Works immediately (zero-shot)
│  ├─ Easily interpretable detections
│  ├─ Transferable to other domains
│  ├─ Actively maintained by Ultralytics
│  └─ Multiple model sizes (nano→large)
└─ Deployment:
   ├─ nano (5-10ms):   Real-time inference
   ├─ small (10-20ms): Better accuracy
   ├─ medium (20-40ms): High accuracy
   └─ large (40-80ms): Maximum accuracy

⚙️ CONFIGURATION
═══════════════════════════════════════════════════════════════════════════════

Modify CONFIG in uav_env.py to customize YOLO:

# Model size selection
'yolo_model_name': 'n',  # Options: 'n', 's', 'm', 'l', 'x'
  n (nano):    5-10ms   per frame - fastest, training recommended
  s (small):   10-20ms  per frame - good balance
  m (medium):  20-40ms  per frame - better accuracy
  l (large):   40-80ms  per frame - highest accuracy

# Detection threshold
'yolo_confidence_threshold': 0.5,  # [0.0, 1.0]
  Lower = more detections (more false positives)
  Higher = fewer detections (fewer false negatives)

# Device selection
'yolo_device': 'cuda',  # 'cuda' for GPU, 'cpu' for CPU
  'cuda': 5-10x faster inference
  'cpu': Slower but no VRAM limitations

🎓 UNDERSTANDING YOLO FEATURES
═══════════════════════════════════════════════════════════════════════════════

Each detection in the 64D vector contains:

Distance Normalization:
  feature_value = (distance_meters / 2.9) → [0.0, 1.0]
  • 0.0 = Very close (0.1m)
  • 0.5 = Mid-range (1.45m)
  • 1.0 = Far away (2.9m)

Angle Normalization:
  feature_value = (angle_degrees + 45) / 90 → [0.0, 1.0]
  • 0.0 = Left (-45°)
  • 0.5 = Center (0°)
  • 1.0 = Right (+45°)

Confidence:
  feature_value = confidence_score → [0.0, 1.0]
  • 1.0 = High confidence detection
  • 0.5 = Medium confidence
  • 0.0 = Low confidence

Detected Flag:
  feature_value ∈ {0.0, 1.0}
  • 1.0 = Object detected
  • 0.0 = No detection

🚨 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Error: "ModuleNotFoundError: No module named 'ultralytics'"
Solution: pip install ultralytics

Error: "CUDA out of memory"
Solution: Use YOLOv8n (nano) or switch to CPU mode

Error: "RuntimeError: MuJoCo camera does not exist"
Solution: Ensure environment.xml has <camera name="uav_depth_camera"/>

Error: "All YOLO features are zero"
Solution: Check RGB→depth conversion, increase confidence threshold

Error: "YOLOv8 model download fails"
Solution: 
  python -c "from ultralytics import YOLO; YOLO('yolov8n.pt')"
  This will download the model

Training is slower than before:
Solution: Use faster YOLOv8 model (nano recommended for training)
