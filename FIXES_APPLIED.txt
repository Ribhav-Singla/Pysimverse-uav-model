ISSUE 3: Observation Space Indexing in RDR Functions
====================================================
Problem:
  - RDR (Ripple Down Rules) system tried to extract depth features from observation using wrong indices
  - Observation space changed from 25D to 142D but code still assumed old layout
  - Caused KeyError when accessing obs[9:9+CONFIG['depth_features_dim']]

Current Observation Layout (142D):
  - [0:3]   = position (3D)
  - [3:6]   = velocity (3D)
  - [6:9]   = goal_distance (3D)
  - [9:137] = CNN depth features (128D)
  - [137:142] = navigation features (5D)
  
Solution Applied:
  - Fixed all RDR depth feature extraction to explicitly index [9:137]
  - Added clear documentation of observation space layout
  - Lines modified: 694, 839, 1109

Files Modified:
  - /uav_env.py:
    - symbolic_action() function (line 694-700)
    - has_specific_rdr_rule() function (line 839-845)
    - _get_reward_and_termination_info() function (line 1109-1115)


ISSUE 4: Missing Variant-Specific PPO Hyperparameters
====================================================
Problem:
  - All three PPO variants (Vanilla, AR, NS) used identical hyperparameters
  - Different PPO types require different learning rates, clip ranges, and optimization epochs
  - User commented that different PPO types should have different parameters
  
Solution Applied:
  - Created separate hyperparameter configurations for each PPO variant
  - Applied variant-specific tuning:
  
  VANILLA PPO (Basic RL):
    - K_epochs: 11
    - eps_clip: 0.1
    - gamma: 0.999
    - lr_actor: 0.00005
    - lr_critic: 0.0002
    - action_std: 1.0
    
  AR PPO (Augmented Rewards):
    - K_epochs: 13 (more epochs for multiple reward signals)
    - eps_clip: 0.1
    - gamma: 0.9995 (higher for reward composition)
    - lr_actor: 0.00006 (slightly higher)
    - lr_critic: 0.00025 (slightly higher)
    - action_std: 1.0
    
  NS PPO (Neurosymbolic):
    - K_epochs: 15 (more epochs for RDR rule refinement)
    - eps_clip: 0.15 (higher for RDR stability)
    - gamma: 0.999
    - lr_actor: 0.00004 (lower for symbolic stability)
    - lr_critic: 0.00015 (lower for symbolic stability)
    - action_std: 0.8 (lower for more focused exploration)

Files Modified:
  - /training.py (lines 35-90): Complete restructure of hyperparameter initialization
