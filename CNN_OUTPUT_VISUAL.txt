CNN OUTPUT & TRAINING FLOW: Visual Explanation
===============================================


1. CNN OUTPUT - WHAT IT PRODUCES
═════════════════════════════════

Input Depth Image:
┌──────────────────────────────────┐
│ 64x64 grayscale pixels           │
│ Value range: 0.1m to 2.9m        │
│                                  │
│ White pixels = close (obstacle)  │
│ Black pixels = far (empty space) │
│ Gray pixels = medium distance    │
└──────────────────────────────────┘
         ↓
    SimplDepthCNN
         ↓
Output: 128-dimensional Feature Vector
┌──────────────────────────────────┐
│ [0.23, -0.15, 1.42, 0.08, ...,  │
│  -0.56, 2.31, ..., -0.33]       │
│                                  │
│ 128 raw numbers (features)       │
│ NOT: Labels or classifications   │
│ NOT: "Obstacle: YES/NO"          │
│ NOT: "Goal: North/South/East"    │
│ NOT: "Boundary: 0.5m away"       │
│                                  │
│ Instead: Abstract representations│
│ that CNN learns are useful       │
└──────────────────────────────────┘

4. REWARD SIGNALS DRIVE LEARNING
═════════════════════════════════

How rewards provide implicit supervision:

Scenario A: Agent moves forward into obstacle
  ├─ CNN features: [high brightness on left sensor]
  ├─ PPO outputs: [forward velocity]
  ├─ Result: Collision!
  ├─ Reward: -5.0 (collision penalty)
  └─ Learning: CNN learns "high brightness = bad"
              PPO learns "don't go forward with this signal"

Scenario B: Agent turns away from obstacle
  ├─ CNN features: [moderate brightness on left]
  ├─ PPO outputs: [turn right velocity]
  ├─ Result: Obstacle avoided!
  ├─ Reward: +0.5 (progress + avoidance)
  └─ Learning: CNN & PPO reinforce this behavior

Scenario C: Agent moves toward goal
  ├─ CNN features: [goal visible in depth]
  ├─ PPO outputs: [move toward goal]
  ├─ Result: Getting closer!
  ├─ Reward: +10 * (distance_reduction)
  └─ Learning: CNN learns "goal pattern = good"
              PPO learns "move toward these features"

Scenario D: Agent reaches goal
  ├─ Distance < 0.1m
  ├─ Reward: +100 (goal reached!)
  └─ Learning: Maximum positive reinforcement!

6. HOW CNN LEARNS GOAL DETECTION
════════════════════════════════════

Goal is ALWAYS at same location: [3, 3, 1]

Early training:
  Agent wanders randomly
  Occasionally stumbles on goal
  Reward: +100 (goal reached!)
  
Mid training:
  Agent learns goal is in that direction
  CNN learns: "Depth pattern near goal location"
  PPO learns: "When these features → move there"
  
Late training:
  Agent actively seeks goal
  CNN recognizes goal area
  PPO navigates toward it
  
Result:
  No explicit "goal location" label
  CNN learns from reward signal
  Behavior emerges: Goal seeking!


7. HOW CNN LEARNS BOUNDARY DETECTION
════════════════════════════════════════

Boundaries are at ±4m in X/Y, 0.3-2.0m in Z

Early training:
  Agent crashes into boundaries frequently
  Reward: -100 (out of bounds!)
  
Mid training:
  Agent learns to stay away from edges
  CNN learns: "Far depth pattern = boundary"
  PPO learns: "When boundary features → turn away"
  
Late training:
  Agent smoothly respects all boundaries
  CNN recognizes boundary areas
  PPO avoids them
  
Result:
  No explicit "boundary distance" label
  CNN learns from penalty signal
  Behavior emerges: Boundary respect!


8. THE MAGIC: NO LABELS NEEDED!
═══════════════════════════════════

Traditional Supervised Learning:
  Need:   [depth_image] → [Label: "obstacle at 0.5m"]
  Problem: Requires manual labeling of thousands of images
  
Reinforcement Learning (Our Approach):
  Need:   [depth_image] → [Action] → [Reward]
  Benefit: Rewards are AUTOMATIC from environment!
  
Key Insight:
  Reward signal = implicit supervision!
  
  Collision → negative reward → CNN learns "this depth = bad"
  Goal reached → positive reward → CNN learns "this depth = good"
  Progress → continuous reward → CNN learns "this direction = correct"
  
Result:
  Learning without labeled data
  CNN learns task-relevant features
  More natural and robust learning
